Analytics Links

      +
      The Analytics Workbench enables you to create and manage remote links.

      In the Analytics Workbench, you can create remote links. These enable you to access data from outside the local Couchbase cluster.

      Permissions Required

      To use all of the functionality in the Analytics Workbench, you must have the Project Owner or Cluster Data / Reader project role.

      View Links

      Any existing links (local and remote) are shown inside the Analytics Scopes working area. This includes information about the type of link, their connection status, any added collections, and a Trash button that allows you to drop each remote link.

      It is not possible to modify an existing link. If you want to modify an existing link, you must delete it and recreate it.

      Create an Amazon S3 Link

      This feature is only available on clusters using Couchbase Server 7.0 or later.

      1. To create an S3 link, click the Add Remote Link button (+) near the top of the Analytics Scopes working area.

        The Add Links flyout is shown.

      2. In the Name field, enter a name for the link.

      3. In the Access Key ID field, enter the access AWS key ID you are using to access Amazon S3.

      4. In the Secret Access Key field, enter the secret access key you are using to access Amazon S3.

      5. Using the Region drop-down menu, choose the AWS region where your S3 bucket resides.

      6. (Optional) In the Service Endpoint field, you can enter a service endpoint that you want to connect to rather than the default S3 endpoint.

      7. Click Create to create the link.

        The new link will appear within the Analytics Scopes working area.

      If you close the Links flyout without creating the link, any information you have entered in the Add Link flyout is not preserved.

      When creating a link to the Amazon S3 service, be sure to follow best practices for security. AWS root account credentials should never be used. The policy for the created IAM User roles should be as strict as possible and only allow access to the required data and required resources. You only need to know the access key ID and the secret access key for the created IAM User role to access the S3 service. The link will be able to access whatever is permitted to the IAM User, since it will be using the IAM User credentials to interact with the AWS S3 service.
